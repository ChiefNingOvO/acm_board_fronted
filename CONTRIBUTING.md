# Contributing

感谢你对这个项目感兴趣。

这个仓库主要面向 ACM / ICPC 比赛大屏场景，因此在提交改动时，请尽量优先考虑下面几点：

- 展示效果是否稳定
- 实时消息链路是否可靠
- 配置项是否清晰、可部署
- 现有大屏布局是否被意外破坏

## 开始之前

建议先完成以下步骤：

1. Fork 本仓库
2. 创建一个新的分支
3. 安装依赖并确保本地可以正常启动

```bash
npm install
npm run proxy:kafka
npm run dev
```

## 提交内容建议

欢迎提交的内容包括：

- Bug 修复
- 样式与交互优化
- 部署文档改进
- 环境配置整理
- Kafka / WebSocket 链路兼容性优化
- 榜单、首刀、判题流的展示逻辑优化

## 提交前检查

提交前请至少确认：

```bash
npm run lint
npm run build
```

如果改动涉及：

- HTTP 接口
- Kafka 消息格式
- 部署文件
- `.env` 配置

请在 PR 描述里说明变更点和影响范围。

## 代码风格

- 使用 TypeScript
- 尽量保持现有文件结构和命名风格
- 配置项优先放进 `.env`
- 避免把真实服务器地址、账号、敏感信息提交到仓库

## Pull Request 建议说明

建议在 PR 中包含：

- 改动背景
- 改了什么
- 是否影响部署
- 是否需要同步修改后端
- 是否有截图或录屏

## Issue 建议

提 Issue 时可以尽量补充：

- 复现步骤
- 期望行为
- 实际行为
- 控制台报错
- 当前接口返回或 Kafka 消息样例

感谢你的贡献。
